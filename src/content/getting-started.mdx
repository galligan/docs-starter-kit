# Getting Started

Get your documentation site up and running with Docs Starter Kit in just a few minutes.

## Prerequisites

Before you begin, make sure you have the following installed:

  <CardContent className="pt-6">
    - **Node.js 18.17** or later
    - **pnpm** (recommended) or npm/yarn
    - **Git** for version control
    - A code editor (VS Code recommended)
  </CardContent>
</Card>

## Project Structure

Once installed, your project will have the following structure:

```
docs-starter-kit/
├── src/
│   ├── app/                 # Next.js App Router
│   │   ├── layout.tsx       # Root layout
│   │   └── [[...slug]]/     # Dynamic routes
│   ├── content/             # Your MDX content
│   │   ├── _meta.json       # Navigation config
│   │   └── *.mdx            # Documentation pages
│   ├── components/          # React components
│   │   └── ui/              # shadcn/ui library
│   ├── lib/                 # Utilities
│   └── mdx-components.tsx   # MDX mappings
├── next.config.mjs          # Next.js config
├── tailwind.config.js       # Tailwind config
└── package.json             # Dependencies
```

## Running the Development Server

Start the development server with Turbopack:

```bash
pnpm run dev
```

<Alert className="mt-4">
  <AlertTitle>🚀 Turbopack Enabled</AlertTitle>
  <AlertDescription>
    The development server uses Turbopack for faster builds and hot reload.
  </AlertDescription>
</Alert>

Your documentation site will be available at [http://localhost:3000](http://localhost:3000).

## Creating Your First Page

1. Create a new MDX file in `src/content/`:

```mdx filename="src/content/my-page.mdx"
# My First Page

This is my first documentation page using Docs Starter Kit!

<Alert>
  <AlertTitle>Hello!</AlertTitle>
  <AlertDescription>
    I can use shadcn/ui components directly in MDX!
  </AlertDescription>
</Alert>
```

2. Update `src/content/_meta.json` to add your page to the navigation:

```json filename="src/content/_meta.json"
{
  "index": {
    "title": "Home",
    "type": "page"
  },
  "my-page": {
    "title": "My Page",
    "type": "page"
  }
}
```

3. Save and see your page appear in the navigation!

## Customizing the Theme

### Update Site Metadata

Edit `src/app/layout.tsx` to customize your site's metadata:

```tsx filename="src/app/layout.tsx"
export const metadata = {
  title: 'My Documentation',
  description: 'Documentation for my awesome project',
}

const navbar = (
  <Navbar
    logo={<b>My Docs</b>}
    projectLink="https://github.com/my-username/my-project"
  />
)
```

### Customize Colors

The site uses Tailwind CSS v4 with OKLCH colors. Edit `src/app.css` to customize:

```css filename="src/app.css"
:root {
  --primary: oklch(0.216 0.006 56.043);
  --background: oklch(1 0 0);
  /* ... other colors */
}
```
