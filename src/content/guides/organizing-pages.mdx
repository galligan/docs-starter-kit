# Organizing Pages

Learn how to structure your documentation with Nextra's file-based routing and navigation system.

## File-Based Routing

Nextra uses a file-system based router. Files in `src/content/` automatically become routes:

<Card className="mt-6">
  <CardContent className="pt-6">
    ```
    src/content/
    ├── index.mdx           → /
    ├── getting-started.mdx → /getting-started
    ├── guides/
    │   ├── _meta.json
    │   ├── intro.mdx       → /guides/intro
    │   └── advanced.mdx    → /guides/advanced
    └── api/
        ├── _meta.json
        └── reference.mdx   → /api/reference
    ```
  </CardContent>
</Card>

## Navigation with _meta.json

The `_meta.json` file controls navigation structure and page metadata:

### Basic Navigation

```json filename="_meta.json"
{
  "index": "Home",
  "getting-started": "Getting Started",
  "guides": "Guides",
  "api": "API Reference"
}
```

### Advanced Configuration

```json filename="_meta.json"
{
  "index": {
    "title": "Home",
    "type": "page",
    "display": "hidden"  // Hide from sidebar
  },
  "guides": {
    "title": "Guides",
    "type": "page"
  },
  "-- Divider": {
    "type": "separator",
    "title": "Resources"
  },
  "github": {
    "title": "GitHub →",
    "type": "page",
    "href": "https://github.com",
    "newWindow": true
  }
}
```

## Organizing Complex Documentation

### Example Structure

<Card className="mt-6">
  <CardContent className="pt-6">

    ```
    src/content/
    ├── _meta.json              # Root navigation
    ├── index.mdx               # Home page
    ├── getting-started.mdx     # Single page
    │
    ├── guides/                 # Guide section
    │   ├── _meta.json
    │   ├── basics/
    │   │   ├── _meta.json
    │   │   ├── installation.mdx
    │   │   └── configuration.mdx
    │   └── advanced/
    │       ├── _meta.json
    │       ├── optimization.mdx
    │       └── deployment.mdx
    │
    ├── api/                    # API docs
    │   ├── _meta.json
    │   ├── components/
    │   │   ├── _meta.json
    │   │   ├── button.mdx
    │   │   └── card.mdx
    │   └── hooks/
    │       ├── _meta.json
    │       └── use-theme.mdx
    │
    └── examples/               # Examples
        ├── _meta.json
        └── blog.mdx
    ```
  </CardContent>
</Card>

### Root _meta.json

```json filename="src/content/_meta.json"
{
  "index": {
    "title": "Documentation",
    "type": "page"
  },
  "getting-started": "Getting Started",
  "guides": "Guides",
  "api": "API Reference",
  "examples": "Examples",
  "-- Support": {
    "type": "separator"
  },
  "github": {
    "title": "GitHub →",
    "href": "https://github.com/...",
    "newWindow": true
  },
  "discord": {
    "title": "Discord →",
    "href": "https://discord.gg/...",
    "newWindow": true
  }
}
```

## Best Practices

<div className="grid grid-cols-1 md:grid-cols-2 gap-4 mt-6">
  <Card>

    </CardHeader>

      - Use clear, descriptive file names
      - Keep nesting to 2-3 levels max
      - Use index.mdx for section overviews

  <Card>
    <CardHeader>
      <CardTitle>🏷️ Naming</CardTitle>
    </CardHeader>
    <CardContent>
      - Use kebab-case for files
      - Be consistent with naming
      - Avoid special characters
      - Keep URLs short and readable
    </CardContent>
  </Card>
  
  <Card>
    <CardHeader>
      <CardTitle>🧭 Navigation</CardTitle>
    </CardHeader>

      - Test navigation on mobile
