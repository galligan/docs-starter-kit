# Writing Content

Learn how to write effective documentation using MDX with Nextra and shadcn/ui components.

## MDX Basics

MDX lets you use JSX components in your markdown content. This means you can write regular markdown and embed React components wherever needed.

### Basic Markdown Syntax

  <CardHeader>

## Using Components

You can use any shadcn/ui component directly in your MDX files:

<Tabs defaultValue="example" className="mt-6">

    <TabsTrigger value="example">Example</TabsTrigger>
    <TabsTrigger value="code">MDX Code</TabsTrigger>
  </TabsList>
  
  <TabsContent value="example">
    <Alert>
      <AlertTitle>Pro Tip</AlertTitle>
      <AlertDescription>
        Components are pre-imported in MDX files - just use them!
      </AlertDescription>
    </Alert>

## Code Blocks

Nextra provides syntax highlighting for code blocks:

### Basic Code Block

```javascript
function greet(name) {
  console.log(`Hello, ${name}!`);
}

greet('World');
```

### Code Block with Title

```typescript filename="utils/greeting.ts"
export function greet(name: string): void {
  console.log(`Hello, ${name}!`);
}
```

### Code Block with Line Highlighting

```jsx {3-5}
function MyComponent() {
  return (
    <Card>
      <CardTitle>Hello</CardTitle>
      <CardContent>World</CardContent>
    </Card>
  );
}
```

## Best Practices

<div className="space-y-4 mt-6">
  <Card>
    <CardHeader>
      <CardTitle>✅ Do</CardTitle>
    </CardHeader>
    <CardContent>
      - Use clear, descriptive headings
      - Break content into logical sections
      - Include code examples
      - Use components to enhance readability
      - Add alt text to images
    </CardContent>
  </Card>
  
  <Card>
    <CardHeader>
      <CardTitle>❌ Don't</CardTitle>
    </CardHeader>
    <CardContent>
      - Create walls of text
      - Skip heading levels (h1 → h3)
      - Overuse components
      - Forget to test your MDX
      - Use unclear link text like "click here"
    </CardContent>
  </Card>
</div>
